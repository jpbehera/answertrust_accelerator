Trust Stack — Microsoft Services Powering AnswerTrust

WHO DOES WHAT
Four capability lanes · every service plugs into the same trace fabric → AnswerLedger → AnswerTrust score. Click any tile for what it does.
📡
Observability
see every hop · every answer
📊
App Insights · OTel
W3C traceparent · GenAI spans
App Insights (OTel) — captures every span: intent → retrieval → model → DLP → red-team. One trace_id stitches user, agent, and data hops.
📒
Real-Time Hub · KQL
AnswerLedger · sub-sec queries
Eventhouse / KQL DB — one row per answer, append-only, queryable in milliseconds. Powers the Trust Dashboard.
🌊
Eventstream · RT Dashboard
live ingest · drill-to-trace · alerts
Eventstream + RT Dashboard + Azure Monitor — streams OTel spans into the AnswerLedger, powers live KPI tiles, and fires alerts on AnswerTrust / DQ / latency drift.
⚖️
Governance
labels · lineage · ownership
🏷️
Microsoft Purview
sensitivity · DLP · catalog · lineage
Purview — sensitivity labels propagate from source to answer; DLP for OneLake blocks egress; catalog gives every dataset an owner.
🆔
Entra ID · OBO
on-behalf-of identity end-to-end
Entra ID — On-Behalf-Of token flow carries the human identity through agent → retriever → data plane. RBAC enforced at source.
🗂️
Fabric Workspaces · OneLake
domain isolation · Azure Policy gates
OneLake + Workspaces + Azure Policy — domain-aligned isolation, shortcuts instead of copies, and Policy enforces DLP / OBO / evaluator config across every Foundry project.
🛡️
Security
block · detect · respond
🚨
Defender for Cloud · AI
runtime threat detection on agents
Defender for Cloud (AI workloads) — detects prompt-injection, data-exfil, jailbreak attempts at the model edge.
🧠
Microsoft Sentinel
SIEM · agent abuse correlation
Sentinel — correlates AnswerLedger events with sign-in/Purview/Defender signals for SOC investigation and SOAR playbooks.
⚔️
Foundry Red-Team · IRM · Key Vault
PyRIT · insider risk · secrets
Red-Team + Purview IRM/DSPM + Key Vault — adversarial suites pre-deploy and in prod, insider-risk + data-exposure signals, and centrally managed model keys / secrets with rotation.
🧪
Quality & Eval
measure · catch · improve
🔍
Fabric DQ Notebook
Validity · Completeness · Freshness
DQ Notebook — runs rules per source table, writes pass-rates to dq_runs, gates retrieval when below threshold.
🎯
Foundry Evaluators
groundedness · relevance · safety
Foundry Evaluators — built-in + custom evaluators score every answer for groundedness, relevance, harmfulness.
📚
Golden-set · AnswerTrust · azd
PR-gated regression · composite score · 1-cmd deploy
Golden-set harness + AnswerTrust composite + azd/Bicep/Actions — versioned evals block PRs on score drop; azd up bootstraps Fabric + Foundry + Purview + dashboards with CI eval gates.
